## Anthropic Haiku 4.5 단가 (기준)

- 입력: **$1.00 / M tokens**
- 출력: **$5.00 / M tokens**
- 캐시 write (prompt cache): +25% 위 입력 단가
- 캐시 read: 10% 위 입력 단가
- **Batch API**: 입력·출력 모두 50% 할인

이번 계산에서 "캐시 read" 비율은 시스템 프롬프트가 매 호출 반복되는 스텝에서 90% 캐시 히트 가정. 첫 호출은 write, 이후는 read.

## 실제와 다르게 산정할 수 있는 지점

1. **캐시 히트율 90% 가정** — 실측 아직. 프롬프트 텍스트 변경 없으면 초당 호출률에 따라 히트율 달라짐. 캐시 read 비율이 60%로 낮아지면 summarize 비용 ~$4.5로 상승.
2. **Pass B 컨텍스트 24K** — 지난 30일 기사 요약 리스트가 이 정도. 아카이브가 커지면 30일 요약도 커져서 컨텍스트 증가. 6개월 시점 재측정 필요.
3. **DAILY_CAP 120** — 현재 실측 일평균 48건이라 CAP 도달 안 함. CAP 도달하면 summarize + F13/F14 대상 기사 증가로 비용 비례.

## 완화 조치

### 이미 적용
- Batch API 50% 할인 (summarize, predict_extract Pass A, model_facts)
- `importance >= 3` 필터로 Pass A/model_facts 후보 축소 (모든 기사 대상 X)
- Pass B는 `_pending_needs_resolution`로 호출 대상 축소 (horizon 지남 OR 60일 초과만)
- 시스템 프롬프트 `cache_control: ephemeral` 적용
- `quarterly_report --auto`로 idempotent (분기 1회만)
