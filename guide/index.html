01프로젝트 위치와 기본 개념

이 저장소는 두 개의 큰 부분으로 나뉩니다.

pipeline/ (Python)

매일 뉴스를 수집·요약·분석해서 data/에 JSON을 씁니다. Python으로 실행.

site/ (Astro + Node)

data/를 읽어서 정적 HTML을 만듭니다. Node/npm으로 실행.

모든 경로는 저장소 최상위(C:\workspace\ai-daily-news\) 기준입니다. cd site라고 쓰여있으면 저장소 최상위에서 site로 들어가라는 뜻입니다.

터미널 위치 확인

Windows PowerShell을 열면 보통 C:\Users\사용자명에서 시작합니다. 먼저 저장소로 이동하세요:

PS> cd C:\workspace\ai-daily-news

이후 프롬프트가 PS C:\workspace\ai-daily-news>로 바뀌면 준비 완료. 어디에 있는지 헷갈리면 언제든 pwd(present working directory)를 치세요.

팁 — 문서에서 cd site && npm run build처럼 &&가 있어도 PowerShell 5.1에서는 &&가 안 됩니다. 대신 두 줄로 나누거나 ;로 연결하세요.

02사이트 빌드 · 개발 (site/)

사이트 관련 명령은 반드시 site/ 폴더 안에서 실행합니다.

처음 한 번: 의존성 설치

PS C:\workspace\ai-daily-news> cd site
PS C:\workspace\ai-daily-news\site> npm install

개발 서버 (브라우저에서 실시간 미리보기)

PS C:\workspace\ai-daily-news\site> npm run dev
# http://localhost:4321/ 접속
# 종료: Ctrl+C

정적 빌드 (배포용)

PS C:\workspace\ai-daily-news\site> npm run build
# 결과: site/dist/ 폴더에 HTML 파일들
# GitHub Actions가 push 이후 자동으로 동일 빌드를 돌립니다

빌드 미리보기 (dist를 로컬에서 확인)

PS C:\workspace\ai-daily-news\site> npm run preview
루트에서 실행하고 싶다면 — npm --prefix site run build로 site/를 지정하면 cd 없이도 됩니다.

03파이프라인 (pipeline/)

파이프라인은 저장소 최상위에서 Python 모듈 형태로 실행합니다. cd pipeline은 하지 않습니다.

처음 한 번: 의존성 설치

PS C:\workspace\ai-daily-news> pip install -e .

전체 파이프라인 순차 실행 (배치 파일)

PS C:\workspace\ai-daily-news> .\run-pipeline.bat
# collect → dedupe → summarize → rank → digest → aggregate_tags
#   → trending → entity_index → themes → predictions
#   → model_facts → quarterly_report --auto → index_latest → build_db

개별 단계

PS C:\workspace\ai-daily-news> python -m pipeline.collect
PS C:\workspace\ai-daily-news> python -m pipeline.dedupe --day 2026-06-30
PS C:\workspace\ai-daily-news> python -m pipeline.summarize --dry-run

드라이런 (LLM 호출 없이 검증)

거의 모든 파이프라인 스텝이 --dry-run을 지원합니다. LLM 비용 없이 후보 개수만 확인하고 싶을 때:

PS C:\workspace\ai-daily-news> python -m pipeline.themes --dry-run
PS C:\workspace\ai-daily-news> python -m pipeline.predict_extract --dry-run
PS C:\workspace\ai-daily-news> python -m pipeline.model_facts --dry-run
PS C:\workspace\ai-daily-news> python -m pipeline.quarterly_report --quarter 2026-Q2 --dry-run

04검증 도구 (npm run verify:*)

N6에서 만든 합성 픽스처 검증과 X2에서 만든 실데이터 스모크가 있습니다. 둘 다 site/에서 npm으로 실행하지만 실제 스크립트는 scripts/에 있습니다.

합성 픽스처 검증 (스키마 드리프트 조기 감지)

PS C:\workspace\ai-daily-news\site> npm run verify:populated
# fixtures/*.json을 잠시 data/에 덮어씌운 후 빌드
# 10개 어서션을 dist HTML에서 확인하고 원본 복원
# 실행 후에도 실 데이터는 그대로

실데이터 스모크 (CI 첫 런 이후)

PS C:\workspace\ai-daily-news\site> npm run verify:realdata
# data/의 실제 파일을 그대로 사용
# 파일이 없으면 SKIP, 있으면 렌더 마커를 확인
주의 — verify:populated가 실행 도중 강제 종료돼도 finally 블록이 원본을 복원합니다. 그래도 걱정되면 실행 전 git status로 data/ 상태를 기억해두세요.

05감사 도구 (audit_*)

세 개의 감사 스크립트가 있습니다. 셋 다 저장소 최상위에서 실행합니다.

클러스터 병합 감사 (P1a / N3 · N5)

PS C:\workspace\ai-daily-news> python -m pipeline.audit_cluster_merge
# reviews/cluster-merge-audit-YYYY-MM-DD.md 자동 생성
# day_span 분포 + 상위 클러스터 + N3 Hamming 히스토그램 + N7 same-day 밴드

소스 헬스 감사 (X4)

PS C:\workspace\ai-daily-news> python -m pipeline.audit_sources
# 3일 연속 0건 소스 리스트
# 매주 월요일 UTC 03:00에 CI가 자동으로 실행 + 이상 시 이슈 생성

파일 카운트 개요 (N7)

PS C:\workspace\ai-daily-news> node scripts/overview-counts.mjs
# 페이지·컴포넌트·파이프라인 모듈·데이터 파일 카운트 표

URL 슬러그 백필 (X1)

PS C:\workspace\ai-daily-news> python -m pipeline.backfill_first_url_hash --dry-run
# cluster_continuity.json에 first_url_hash를 채우는 일회성 스크립트
# --dry-run 없이 실행하면 실제로 씀

06Git 배포 흐름

커밋 → push하면 GitHub Actions가 자동으로 빌드하고 배포합니다.

1

변경 상태 확인

PS C:\workspace\ai-daily-news> git status
PS C:\workspace\ai-daily-news> git diff
2

스테이징 (특정 파일만 권장)

PS C:\workspace\ai-daily-news> git add pipeline/foo.py site/src/pages/bar.astro
3

커밋 + push

PS C:\workspace\ai-daily-news> git commit -m "feat: 짧은 설명"
PS C:\workspace\ai-daily-news> git push
4

배포 확인 — 2~3분 후 사이트에서 반영 확인. Actions 탭에서 진행 상황도 볼 수 있습니다.

중요 — .env 같은 secrets 파일은 절대 커밋하지 마세요. gitleaks가 push마다 자동 스캔합니다.

07자주 하는 실수

증상원인 · 해결
npm: command not found Node.js가 설치 안 됨. nodejs.org에서 LTS 설치.
Cannot find module '~/lib/...' cd site를 안 하고 최상위에서 실행. site 폴더 안에서 실행해야 합니다.
python: command not found Python 3.11 설치 후 py가 아니라 python이 되는지 확인. 안 되면 py -m pipeline.collect로 대체.
ANTHROPIC_API_KEY 없음 저장소 최상위에 .env 파일 만들고 ANTHROPIC_API_KEY=sk-... 입력. .env.example 참고.
PowerShell에서 &&가 안 됨 PS 5.1 한계. 두 명령으로 나누거나 ;로 연결.
커밋 메시지에 한글이 깨짐 커밋 메시지를 파일에 저장 후 git commit -F .commitmsg로 전달. -m "한글"은 인코딩 이슈 가능.
사이트 배포 후 반영 안 됨 GitHub Actions 탭에서 deploy.yml이 실패했는지 확인. 보통 2~3분 후 반영.
